## 目標
LangChain Ch4〜5 の `ChatPromptTemplate` / `ChatOpenAI` を使い、
現在 `openai_adapter.py` で直接扱っている OpenAI SDK を LangChain ベースに移行する。
後の STEP3（感情状態）・STEP4（視聴者記憶）でプロンプトへ変数を差し込める土台を作る。

---

## 作業内容

1. **依存パッケージ追加** (`requirements.txt`)
   - `langchain>=0.3` を追記
   - `langchain-openai>=0.2` を追記
   - `pip install -r requirements.txt` でインストールを確認する

2. **`core/langchain_adapter.py` を新規作成**
   以下の仕様で実装する：
   - `ChatOpenAI` をインポートし、`config/settings.yaml` の `llm.model` と `.env` の `OPENAI_API_KEY` / `base_url` で初期化
   - `ChatPromptTemplate.from_messages()` でシステムプロンプトとユーザー入力を構造化する
   - システムプロンプトには将来の変数差し込み用プレースホルダーとして
     `{emotion}` (例: "普通") と `{memory}` (例: "") を含める
   - クラス名は `LangChainAdapter`、公開メソッドは既存の `create_chat(question: str) -> str` と同一シグネチャを維持する
   - 直近 5 件の会話履歴を保持する仕組みは既存 `openai_adapter.py` の実装を踏襲する

3. **システムプロンプトをテンプレート化**
   `core/system_prompt.txt` の内容を `core/system_prompt_template.txt` としてコピーし、
   末尾に以下 2 行を追加する（既存ファイルはそのまま残す）：
   ```
   [現在の感情状態]
   {emotion}
   [記憶している視聴者情報]
   {memory}
   ```

4. **`core/aituber_system.py` の差し替え**
   - `from openai_adapter import OpenAIAdapter` を削除
   - `from langchain_adapter import LangChainAdapter` を追加
   - `self.openai_adapter = OpenAIAdapter()` を `self.langchain_adapter = LangChainAdapter()` に変更
   - `talk_with_comment()` 内の `self.openai_adapter.create_chat(comment)` を
     `self.langchain_adapter.create_chat(comment)` に変更

5. **動作確認**
   `core/run_dummy.py` を実行し、ダミーコメント 5 件すべてで
   LLM 応答 → VOICEVOX 音声合成 → 音声再生が成功することを確認する

## 注意事項

- `openai_adapter.py` は削除せず残しておく（比較・ロールバック用）
- `{emotion}` のデフォルト値は `"普通"`、`{memory}` のデフォルト値は `""` として
  `LangChainAdapter.__init__()` で初期化すること
- STEP3・STEP4 でこれらの値を外から注入できるよう、
  `create_chat(question, emotion="普通", memory="")` のように引数で受け取る設計にすること
