### add_conditional_edges の本質
| 関数 | 役割 |
|---|---|
| `add_edge(A, B)` | AからBへ**常に**遷移する固定エッジ |
| `add_conditional_edges(A, func, map)` | `func` の戻り値によって**遷移先を分岐**させる |

```python
# 例：感情によって応答トーンを変える
graph.add_conditional_edges(
    "analyze_emotion",
    lambda state: state["emotion"],  # ← 戻り値で分岐先を決定
    {
        "嬉しい":  "adjust_cheerful",
        "悲しい":  "adjust_calm",
        "普通":   "adjust_normal",
    }
)
```

このタスクで実際に実装しながら理解を深める。

#### 1-2. 感情別のパラメータ調整ノードを作成
以下の5つのノード関数を作成する：

```python
def adjust_happy(state: EmotionState) -> dict:
    """嬉しい時：明るく速めに話す"""
    return {"speed": 1.2, "pitch": 0.1}

def adjust_excited(state: EmotionState) -> dict:
    """ワクワク時：さらに明るく元気に"""
    return {"speed": 1.3, "pitch": 0.15}

def adjust_surprised(state: EmotionState) -> dict:
    """驚き時：高めの声で"""
    return {"speed": 1.1, "pitch": 0.2}

def adjust_sad(state: EmotionState) -> dict:
    """悲しい時：ゆっくり低めに話す"""
    return {"speed": 0.9, "pitch": -0.05}

def adjust_normal(state: EmotionState) -> dict:
    """普通時：デフォルト"""
    return {"speed": 1.0, "pitch": 0.0}
```

#### 1-3. StateGraph に条件分岐を追加

**現在のグラフ構造（STEP3）**：
```
analyze_emotion → END
```

**変更後のグラフ構造（STEP3.5）**：
```
analyze_emotion → [条件分岐] → adjust_happy / adjust_excited / adjust_surprised / adjust_sad / adjust_normal → END
```

**実装コード**：
```python
graph = StateGraph(EmotionState)

# 既存ノード
graph.add_node("analyze_emotion", analyze_emotion_node)

# 新規ノード（5つ）
graph.add_node("adjust_happy", adjust_happy)
graph.add_node("adjust_excited", adjust_excited)
graph.add_node("adjust_surprised", adjust_surprised)
graph.add_node("adjust_sad", adjust_sad)
graph.add_node("adjust_normal", adjust_normal)

# エントリーポイント（変更なし）
graph.set_entry_point("analyze_emotion")

# ★ 条件分岐エッジ（ここが今回の学習ポイント）
graph.add_conditional_edges(
    "analyze_emotion",
    lambda state: state["emotion"],  # 感情ラベルで分岐先を決定
    {
        "嬉しい": "adjust_happy",
        "ワクワク": "adjust_excited",
        "驚き": "adjust_surprised",
        "悲しい": "adjust_sad",
        "普通": "adjust_normal",
    }
)

# 各調整ノードから END へ（固定エッジ）
graph.add_edge("adjust_happy", END)
graph.add_edge("adjust_excited", END)
graph.add_edge("adjust_surprised", END)
graph.add_edge("adjust_sad", END)
graph.add_edge("adjust_normal", END)

self.app = graph.compile()
```

#### 1-4. EmotionAgent.analyze() の戻り値を拡張
```python
def analyze(self, comment: str) -> dict:
    """
    感情分析 + VOICEVOXパラメータ調整を実行

    Returns:
        {
            "emotion": str,  # 感情ラベル
            "speed": float,  # VOICEVOXのspeed
            "pitch": float   # VOICEVOXのpitch
        }
    """
    result = self.app.invoke({
        "comment": comment,
        "emotion": "",
        "speed": 1.0,
        "pitch": 0.0
    })

    return {
        "emotion": result["emotion"],
        "speed": result["speed"],
        "pitch": result["pitch"]
    }
```

## 注意事項

### VOICEVOX パラメータの範囲
- **speedScale**: 0.5〜2.0（推奨: 0.8〜1.5）
- **pitchScale**: -0.15〜0.15（大きすぎると不自然）

### add_conditional_edges の復習ポイント
- **第1引数**: 分岐元のノード名（`"analyze_emotion"`）
- **第2引数**: 分岐先を決める関数（`lambda state: state["emotion"]`）
- **第3引数**: 戻り値とノード名のマッピング（`{"嬉しい": "adjust_happy", ...}`）

### デバッグ方法
もし分岐がうまく動かない場合：
```python
# 分岐関数にprintを追加してデバッグ
def route_by_emotion(state: EmotionState):
    emotion = state["emotion"]
    print(f"[DEBUG] 分岐先を決定: emotion={emotion}")
    return emotion

graph.add_conditional_edges(
    "analyze_emotion",
    route_by_emotion,  # ← lambda の代わりに名前付き関数を使う
    {...}
)
```

## 学習ポイント（理解度チェックQ3の復習）

このタスクで `add_conditional_edges` を実装することで、以下が理解できる：

| 項目 | 内容 |
|---|---|
| **固定エッジとの違い** | `add_edge` は常に同じ遷移、`add_conditional_edges` は状態によって変わる |
| **分岐の仕組み** | 第2引数の関数が返す値が、第3引数のdictのキーと一致するノードへ遷移 |
| **実用例** | 感情→応答調整、ユーザータイプ→応答戦略、エラー有無→リトライorスキップ 等 |

STEP4（Neo4j視聴者記憶）では「初回訪問 or リピーター」で分岐する際にも使える概念。
