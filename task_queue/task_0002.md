## 目標
LangGraph Ch9 の `StateGraph` を使い、コメントの感情を分析する `EmotionAgent` を実装する。
分析した感情ラベルを `LangChainAdapter.create_chat(emotion=...)` に渡すことで、
ナナカの回答がコメントの雰囲気に合わせて変化するようにする。

## 前提知識の確認（STEP2理解度チェックの補足）
実装前に以下を必ず理解してから進むこと：

- `emotion` は **VOICEVOX のパラメータではない**
- `emotion` は `create_chat()` → `chain.invoke({"emotion": ...})` → `system_prompt_template.txt` の
  `{emotion}` に埋め込まれ → **LLMへの指示（プロンプト）の一部** として機能する
- つまり `emotion="嬉しい"` にすると、LLM が「感情状態：嬉しい」を考慮した回答を生成する
- VOICEVOX はその後に来る「テキスト→音声変換」の層であり、emotion/memory とは無関係

## 作業内容

1. **依存パッケージ追加** (`requirements.txt`)
   - `langgraph>=0.2` を追記する
   - `pip install -r requirements.txt` でインストールを確認する

2. **`agents/emotion_agent.py` を新規作成**
   以下の仕様で実装する：

   **① EmotionState の定義**
   ```python
   from typing import TypedDict

   class EmotionState(TypedDict):
       comment: str   # 分析対象のコメント
       emotion: str   # 分析結果の感情ラベル
   ```

   **② 感情分析ノード**
   - `ChatOpenAI`（settings.yaml / .env を参照）で以下のプロンプトを使う：
     ```
     以下のコメントを読んで、ナナカの感情状態を次の5つから1つだけ選んでください。
     感情ラベルのみを出力し、他の文字は一切含めないでください。
     選択肢: 嬉しい / 普通 / ワクワク / 驚き / 悲しい

     コメント: {comment}
     ```
   - 返答から感情ラベルを抽出して `state["emotion"]` にセットし、state を返す

   **③ StateGraph の構築**
   ```python
   from langgraph.graph import StateGraph, END

   graph = StateGraph(EmotionState)
   graph.add_node("analyze_emotion", <感情分析ノード関数>)
   graph.set_entry_point("analyze_emotion")
   graph.add_edge("analyze_emotion", END)
   self.app = graph.compile()
   ```

   **④ EmotionAgent クラス**
   - クラス名：`EmotionAgent`
   - 公開メソッド：`analyze(comment: str) -> str`
     - `self.app.invoke({"comment": comment, "emotion": ""})` を呼び出す
     - 返り値の `state["emotion"]` を返す
     - LLM の返答が5つのラベル以外だった場合は `"普通"` にフォールバック

3. **`core/aituber_system.py` に統合**
   - `from agents.emotion_agent import EmotionAgent` を追加（sys.path の調整が必要な場合は対処する）
   - `self.emotion_agent = EmotionAgent()` を `__init__()` に追加
   - `talk_with_comment()` 内を以下の順序に変更：
     1. `comment = self.youtube_comment_adapter.get_comment()`
     2. `emotion = self.emotion_agent.analyze(comment)` ← 追加
     3. `print(f"[感情分析] {emotion}")` ← ターミナル確認用に追加
     4. `response_text = self.langchain_adapter.create_chat(comment, emotion=emotion)` ← emotion を渡す

4. **動作確認**
   `core/run_dummy.py` を実行し、以下をすべて確認する：
   - ダミーコメント5件それぞれで `[感情分析] ○○` がターミナルに表示される
   - LLM 応答 → VOICEVOX 音声合成 → 音声再生が成功する（STEP2 と同じ5件）
   - コメントによって感情ラベルが変化していること（全件 "普通" は失敗とみなす）
