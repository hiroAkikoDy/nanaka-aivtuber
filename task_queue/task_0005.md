### 3. `agents/dialogue_coordinator.py` を新規作成

複数ペルソナの会話を調整するコーディネーター：

#### 3-1. DialogueCoordinator クラス
```python
class DialogueCoordinator:
    def __init__(self):
        self.nanaka = PersonaAgent("nanaka")
        self.ryou = PersonaAgent("ryou")
        self.turn_limit = 4  # settings.yaml から読み込み

    def coordinate_dialogue(self, viewer_comment: str, emotion: str) -> list[dict]:
        """
        視聴者コメントを元に、2人のAIが会話を展開

        Args:
            viewer_comment: 視聴者のコメント
            emotion: EmotionAgentで分析された感情

        Returns:
            list[dict]: [
                {"persona": "nanaka", "text": "...", "speaker_id": 46},
                {"persona": "ryou", "text": "...", "speaker_id": 3},
                ...
            ]
        """
        dialogue = []
        context = f"視聴者: {viewer_comment}\n"

        # ターン1: ナナカが最初に応答
        nanaka_response = self.nanaka.respond(context, emotion)
        dialogue.append({
            "persona": "nanaka",
            "text": nanaka_response,
            "speaker_id": self.nanaka.get_speaker_id()
        })
        context += f"ナナカ: {nanaka_response}\n"

        # ターン2〜N: リョウとナナカが交互に会話
        for turn in range(1, self.turn_limit):
            if turn % 2 == 1:  # リョウのターン
                ryou_response = self.ryou.respond(context, emotion)
                dialogue.append({
                    "persona": "ryou",
                    "text": ryou_response,
                    "speaker_id": self.ryou.get_speaker_id()
                })
                context += f"リョウ: {ryou_response}\n"
            else:  # ナナカのターン
                nanaka_response = self.nanaka.respond(context, emotion)
                dialogue.append({
                    "persona": "nanaka",
                    "text": nanaka_response,
                    "speaker_id": self.nanaka.get_speaker_id()
                })
                context += f"ナナカ: {nanaka_response}\n"

        return dialogue
```

### 5. `core/aituber_system.py` の拡張

#### 5-1. マルチペルソナモードの追加
```python
class AITuberSystem:
    def __init__(self):
        # 既存の初期化...

        # マルチペルソナ設定の読み込み
        self.multi_persona_enabled = settings["multi_persona"]["enabled"]

        if self.multi_persona_enabled:
            from agents.dialogue_coordinator import DialogueCoordinator
            self.dialogue_coordinator = DialogueCoordinator()

    def talk_with_comment(self, viewer_name: str, comment: str):
        # 1. 記憶取得（既存）
        memory = self.memory_agent.get_memory(viewer_name)

        # 2. 感情分析（既存）
        emotion_result = self.emotion_agent.analyze(comment)
        emotion = emotion_result["emotion"]

        # 3. マルチペルソナモード分岐
        if self.multi_persona_enabled:
            # マルチペルソナ: 複数AIが会話
            dialogue = self.dialogue_coordinator.coordinate_dialogue(comment, emotion)

            for turn in dialogue:
                print(f"[{turn['persona']}] {turn['text']}")
                self.voicevox_adapter.create_voice(
                    turn['text'],
                    speaker_id=turn['speaker_id'],
                    speed=emotion_result["speed"],
                    pitch=emotion_result["pitch"]
                )

            # 記憶保存: 最後のナナカの発言を保存
            nanaka_turns = [t for t in dialogue if t['persona'] == 'nanaka']
            if nanaka_turns:
                last_nanaka_response = nanaka_turns[-1]['text']
                self.memory_agent.save_interaction(viewer_name, comment, emotion, last_nanaka_response)
        else:
            # 単独モード（既存のロジック）
            response_text = self.langchain_adapter.create_chat(comment, emotion=emotion, memory=memory)
            print(f"[応答] {response_text}")
            self.voicevox_adapter.create_voice(
                response_text,
                speed=emotion_result["speed"],
                pitch=emotion_result["pitch"]
            )
            self.memory_agent.save_interaction(viewer_name, comment, emotion, response_text)
```

## 注意事項

### ペルソナ設計のポイント
- **役割分担を明確に**: ナナカ=栽培、リョウ=料理で専門性を分ける
- **話し方の違い**: 性格が会話に反映されるようにプロンプトを工夫
- **会話の流れ**: 一方的な情報提供にならず、相互に反応し合う

### 会話ターン数の調整
- 短すぎる（1〜2ターン）: 議論が浅い
- 長すぎる（6ターン以上）: 視聴者が飽きる、応答時間が長い
- **推奨**: 4ターン（ナナカ→リョウ→ナナカ→リョウ）

### VOICEVOX speaker_id の選択
- ナナカ: 46（四国めたん）— 既存
- リョウ: 3（ずんだもん）または 8（春日部つむぎ）等
- **確認方法**: `http://localhost:50021/speakers` でspeaker一覧を取得

### Neo4j記憶の扱い
- マルチペルソナモードでは、ナナカの最後の発言を記憶として保存
- リョウの発言は記憶に含めない（ナナカとの一貫性を保つため）

### 将来の拡張性
- STEP6以降: 3人以上のペルソナ（専門家、視聴者代表 等）
- YouTube Live コメントとの連携
- 視聴者が対話相手を指定できる機能

## デバッグのヒント

### 会話が不自然な場合
- `personas.yaml` のシステムプロンプトを調整
- 「他のペルソナの発言を踏まえて応答してください」を追加

### 音声の切り替えがうまくいかない場合
- `voicevox_adapter.py` の `speaker_id` が正しく渡されているか確認
- ログに `[DEBUG] speaker_id={値}` を出力して追跡

### ターン数が想定と異なる場合
- `DialogueCoordinator.coordinate_dialogue()` のループロジックを確認
- `turn_limit` が設定通りか確認
