## 目標
LangChain Ch6 と Neo4j を使い、視聴者ごとの会話履歴を記憶する `MemoryAgent` を実装する。
同じ視聴者が再度コメントした際に「前回〇〇さんが質問していた△△の件ですが」のような
文脈を保った応答ができるようにする。

### 2. `memory/memory_agent.py` を新規作成

以下の仕様で実装する：

#### ① Neo4jドライバの初期化
```python
from neo4j import GraphDatabase
import os
from dotenv import load_dotenv

class MemoryAgent:
    def __init__(self):
        load_dotenv()
        uri = os.getenv("NEO4J_URI")
        user = os.getenv("NEO4J_USER")
        password = os.getenv("NEO4J_PASSWORD")
        self.driver = GraphDatabase.driver(uri, auth=(user, password))
```

#### ② データモデル（Cypherクエリ）
- **Viewerノード**: `(v:Viewer {name: str, first_seen: datetime, last_seen: datetime})`
- **Commentノード**: `(c:Comment {text: str, timestamp: datetime, emotion: str, response: str})`
- **リレーション**: `(v)-[:COMMENTED]->(c)`

#### ③ 公開メソッド

**`save_interaction(viewer_name: str, comment: str, emotion: str, response: str) -> None`**
- Viewer ノードを `MERGE`（存在しなければ作成、あればlast_seenを更新）
- Comment ノードを作成
- `COMMENTED` リレーションを作成

**`get_memory(viewer_name: str) -> str`**
- 同じ Viewer の過去3件のコメントを新しい順に取得
- 以下の形式で文字列を生成して返す：
  ```
  過去の会話履歴:
  - [2026-05-09 10:30] コメント「クレソンの育て方教えて」→ 応答「水が大切なんだよ」
  - [2026-05-08 15:20] コメント「ナナカちゃん可愛い」→ 応答「ありがとう！」
  ```
- 履歴がない場合は空文字列 `""` を返す

#### ④ 例外処理
- Neo4j接続エラー時はログ出力して空文字列を返す（システム全体を停止させない）

**4-4. 処理順序の変更**
```python
def talk_with_comment(self, viewer_name: str, comment: str):
    # 1. 過去の記憶を取得
    memory = self.memory_agent.get_memory(viewer_name)
    print(f"[記憶取得] {viewer_name}: {len(memory)}文字")

    # 2. 感情分析
    emotion = self.emotion_agent.analyze(comment)
    print(f"[感情分析] {emotion}")

    # 3. LLM応答生成（emotion + memory を渡す）
    response_text = self.langchain_adapter.create_chat(
        comment,
        emotion=emotion,
        memory=memory
    )
    print(f"[応答] {response_text}")

    # 4. 音声合成・再生
    self.voicevox_adapter.create_voice(response_text)

    # 5. 会話履歴を保存
    self.memory_agent.save_interaction(viewer_name, comment, emotion, response_text)
    print(f"[記憶保存] {viewer_name} の会話を保存しました")
```

## 注意事項

- **Neo4j のバージョン**: 5.x 系を推奨（開発者の経験年数5年と記載あり）
- **タイムゾーン**: `datetime` は UTC で保存し、表示時に JST に変換すること
- **記憶の粒度**: 初期実装では過去3件に制限（将来的にベクトル検索で拡張可能）
- **エラーハンドリング**: Neo4j接続失敗時もシステム全体は停止させず、memory="" として続行する
- **セキュリティ**: `.env` は `.gitignore` に含まれていることを確認（APIキーと同様）
- **前プロジェクトとの差異**:
  - クレソンAI では Tool Selector があったが、本プロジェクトでは単純な記憶取得のみ
